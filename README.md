

### 注意
- 本项目适合alist挂载网盘的增量更新，监控目录内每增加一个文件，你只需要手动刷新一个网页就可以实现新增文件的重命名，当然也支持全量重命名以及strm生成
- 命名后无法还原更改，所以建议先用小文件夹测试明白后再投入使用
### 本项目功能：
可以对网盘电影，剧集，动漫进行重命名以及strm文件的生成，只要支持挂载到alist的网盘都可以进行重命名（115 阿里云盘 天翼云盘 等）
#### 对电影处理
- 1.删除电影文件夹内的垃圾视频，只保留一个最大的本体视频以及字幕文件，然后将字幕文件命名为电影文件的名字
- 2.将电影文件生成emby可以读取的strm文件，并且将字幕文件下载到本地，以供emby读取直链
- 3.在网盘删除电影后刷新，程序会自动删除相应的strm文件夹，并刷新emby对应的媒体库
#### 对剧集和动漫处理
- 1.新增剧集后，手动访问刷新页面，程序会自动对新增的剧集进行重命名，但是剧集命名要符合规范（剧集的目录结构应该是剧集名字目录（名字要准确，不要有广告年份等字样，否则会影响emby刮削），剧集名字目录下是Season目录，Season内是
相应的视频以及字幕文件，Season目录内小于20m的视频则被视为垃圾文件被删除，同样Season目录下的tar，txt，目录等非视频文件也会被自动删除），剧集目录下除了Season以及Specials的其他目录都会被删除，请小心使用
- 2.剧集内的字幕文件会和剧集的名字一样，命名之后视频文件生成strm，字幕文件直接下载到本地
- 3.剧集命名之后结构如下，命名前也应该按照如此结构放置视频文件
```plaintext
/Media/
├── Movies/
│   └── 阿凡达 (2009)/
│       ├── Avatar.mkv
│       └── Avatar.zh.srt
└── Shows/
    └── 权力的游戏/
        ├── Season 01/
        │   ├── S01E01.mkv
        │   └── S01E01.zh.srt
        └── Season 02/
            └── S02E01.mkv
```


### 使用教程
- 创建文件夹alist-rename-strm，将docker-compose.yaml、config、data下载进去，
最后启动docker-compose up -d
